# bifrost VPS — towonel migration runbook

Host: `bifrost` (debian 12+, public IP w Infisical `/vps/bifrost/VPS_IP`).

## Stack

```
docker/bifrost/
├── 01-crowdsec/        CrowdSec + journald-ssh + GAFAM ASN + geoip-enrich
├── 02-towonel/         towonel-hub (tunnel server + TLS edge + port 22 for forgejo SSH)
├── 03-caddy-l4/        Caddy layer4 (SNI router) + http reverse_proxy with basic_auth for /metrics
├── 04-gatus/           External watchdog + receives cluster heartbeat
├── 05-maxmind/         geoipupdate container — GeoLite2 country/ASN CSV+MMDB
├── 06-monitoring/      node-exporter (scraped by K8s prom via Caddy basic_auth)
└── .doco-cd/           doco-cd self-bootstrap compose

ansible/bifrost/playbook.yaml      Monolithic Ansible bootstrap (no roles)
```

Doco-cd pulls secrets **directly from Infisical** (`SECRET_PROVIDER=infisical`).
The ansible playbook only writes 3 file-mounted secrets needed by doco-cd itself
(infisical client secret, api_secret, webhook_secret). Everything else flows
through doco-cd's `external_secrets` feature in `docker/bifrost/.doco-cd.yaml`.

## Infisical secrets

### `/vps/bifrost`

| Key                            | Purpose                                                                  |
| ------------------------------ | ------------------------------------------------------------------------ |
| `VPS_IP`                       | bifrost public IP (used by K8s DNSEndpoint + cluster gatus)              |
| `INFISICAL_DOCOCD_CLIENT_ID`   | Machine identity for doco-cd to pull secrets                             |
| `INFISICAL_DOCOCD_CLIENT_SECRET` | ↑                                                                       |
| `DOCOCD_API_SECRET`            | Auth for doco-cd API (≥32 char random: `openssl rand -hex 32`)           |
| `DOCOCD_WEBHOOK_SECRET`        | GitHub webhook HMAC secret                                               |
| `CROWDSEC_API_KEY`             | First run: leave empty, paste after `cscli bouncers add`                 |
| `TOWONEL_ACME_EMAIL`           | `admin@vzkn.eu`                                                          |
| `TOWONEL_INVITE_HASH_KEY`      | `openssl rand -hex 32`                                                   |
| `TOWONEL_INVITE_TOKEN`         | Generated by hub after bootstrap (step 4 below) — for K8s towonel-agent  |
| `GATUS_PUSHOVER_TOKEN`         | Pushover application token                                               |
| `GATUS_PUSHOVER_KEY`           | Pushover user key                                                        |
| `BIFROST_HEARTBEAT_TOKEN`      | Shared with cluster gatus: `openssl rand -hex 16`                        |
| `BIFROST_METRICS_BCRYPT`       | bcrypt hash of metrics scrape password (Caddy basic_auth)                |
| `VPS_PROMETHEUS_USER`          | Plain username (must be `prometheus`)                                    |
| `VPS_PROMETHEUS_PASSWORD`      | Plain password — bcrypt this → `BIFROST_METRICS_BCRYPT`                  |

### `/shared/maxmind`

| Key                   | Purpose                          |
| --------------------- | -------------------------------- |
| `MAXMIND_USER_ID`     | MaxMind account ID               |
| `MAXMIND_LICENSE_KEY` | MaxMind license key              |

### Generate `BIFROST_METRICS_BCRYPT`

```bash
PASS="$(openssl rand -base64 32)"
echo "plain: $PASS"  # → VPS_PROMETHEUS_PASSWORD
docker run --rm caddy:2 caddy hash-password --plaintext "$PASS"  # → BIFROST_METRICS_BCRYPT
```

## Cutover

1. **Set all `/vps/bifrost` + `/shared/maxmind` secrets** in Infisical
   (skip `TOWONEL_INVITE_TOKEN` and `CROWDSEC_API_KEY` — those come back in step 4).
2. **Run the playbook** (SSH on port 22 the first time):
   ```bash
   cd ansible/
   ansible-galaxy install -r requirements.yaml
   ansible-playbook -i inventory.yaml bifrost/playbook.yaml
   ```
   After this: SSH on `22222`, doco-cd up, all `docker/bifrost/*` stacks deployed.
3. **Get CrowdSec API key**, write to Infisical, rerun crowdsec tag:
   ```bash
   ssh -p 22222 debian@bifrost.vzkn.eu \
     'docker exec crowdsec cscli bouncers add firewall -o raw'
   # paste into /vps/bifrost/CROWDSEC_API_KEY, then:
   ansible-playbook -i inventory.yaml bifrost/playbook.yaml --tags crowdsec
   ```
4. **Generate towonel-agent invite token** (one-time):
   ```bash
   OP_KEY=$(ssh -p 22222 debian@bifrost.vzkn.eu sudo cat /opt/towonel/operator.key)
   curl -s https://twnl.vzkn.eu/v1/invites \
     -X POST -H "Authorization: Bearer ${OP_KEY}" \
     -d '{"name":"k8s-agent"}' | jq -r .token
   ```
   Paste into `/vps/bifrost/TOWONEL_INVITE_TOKEN`.
5. **K8s side** — `towonel-agent-secret` syncs (every 1m), agent starts and connects.
   - HTTPS for `*.vzkn.eu` → `envoy-external.network.svc.cluster.local:443`
   - SSH on `ssh.git.vzkn.eu:22` → `forgejo-ssh.development.svc.cluster.local:22`
6. **GitHub webhook** (optional, makes deploys instant instead of 1h poll):
   - URL: `https://bifrost-doco-cd.vzkn.eu/v1/webhook`
   - Content type: `application/json`
   - Secret: `$DOCOCD_WEBHOOK_SECRET`
   - Events: just the push event

## Adding new TCP services through towonel

Edit `kubernetes/apps/network/towonel-agent/helmrelease.yaml` →
`TOWONEL_AGENT_TCP_SERVICES`. Allow the port in UFW
(`ansible/bifrost/playbook.yaml` → Firewall block) and expose it on `02-towonel`
(`ports: - <port>:<port>`).

## Observability paths

- **K8s prom → VPS node-exporter**: ScrapeConfig `vps-bifrost` in
  `kube-prometheus-stack/scrapeconfig.yaml`, target `bifrost.vzkn.eu:443/metrics`,
  HTTPS + basic auth.
- **Cluster gatus → VPS**: pings `${VPS_IP}` (icmp) + polls `https://twnl.vzkn.eu` (HTTPS).
  PUSHES heartbeat every 60s to bifrost gatus.
- **VPS gatus → Cluster**: polls `https://status.vzkn.eu` every 60s. Alerts via Pushover
  on its own (independent of cluster Alertmanager).
- **GeoIP blocking**: `geoipupdate` refreshes `/opt/maxmind/GeoLite2-Country-CSV_*/` every 72h.
  Cron (weekly + on reboot) runs `update-country-blocks.sh` which rebuilds
  `countries-blocked-{v4,v6}` ipsets from CSV bundle and links them via `GEO_BLOCK_CHAIN`
  jump in iptables INPUT/FORWARD/DOCKER-USER chains.
